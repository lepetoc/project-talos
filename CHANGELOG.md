# Changelog

All notable changes to this project are documented here.

## 0.1.0

Initial pre-release development.

- Core alarm state machine (`core/`): arming, disarming, entry/exit delays,
  and zone tracking, implemented as dependency-free logic driven entirely by
  the caller.
- HTTP/WebSocket API (`api/`): Axum server, SQLite persistence, JWT-based
  authentication, zone management, and real-time state updates over
  WebSocket.
- SIA DC-09 monitoring-center reporting module, including tests against a
  simulated receiver verifying the wire protocol and acknowledgment handling.
- Shelly BLE/BTHome sensor decoding module (work in progress, unit-tested but
  not yet fully confirmed against real hardware).
- Minimal single-page frontend (Alpine.js, no build step) served directly by
  the API.
